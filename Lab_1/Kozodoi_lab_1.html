У чому відмінність інтерпретатора від компілятора? Python – компілятор чи інтерпретатор?¶

Компілятор перетворює вихідний код на машинний код заздалегідь, тоді як інтерпретатор виконує цю операцію поступово, під час роботи програми. Python є інтерпретованою мовою програмування. Це означає, що код Python виконується безпосередньо інтерпретатором, без необхідності попередньої компіляції

Що таке IDE PyCharm?¶

PyCharm — інтегроване середовище розробки для мови програмування Python. Надає засоби для аналізу коду, графічний зневаджувач, інструмент для запуску юніт-тестів

Що таке система контроля версій (СКВ) і для чого вона призначена? У чому принципова відмінність СКВ SVN та GIT?¶

Система контроля версій (СКВ) – це інструмент, який використовується для управління змінами в коді та інших ресурсах проекту¹. Основні функції СКВ включають:

  • Відстеження змін: зберігання історії змін у файлах.
  • Повернення до попередніх версій: можливість відновити попередні версії файлів.
  • Спільна робота: підтримка командної роботи над проектом.
  • Злиття змін: об'єднання змін з різних гілок¹².##

Принципова відмінність між SVN та Git¶

  • SVN (Subversion):

    • Централізована система: всі дані зберігаються на одному центральному сервері¹.
    • Простота використання: легше налаштувати та використовувати для невеликих проектів¹.
    • Менше конфліктів: оскільки всі зміни проходять через центральний сервер, конфлікти виникають рідше¹.

  • Git:

    • Розподілена система: кожен користувач має повну копію репозиторію¹².
    • Висока швидкість: швидше працює з великими проектами та підтримує ефективне гілкування¹.
    • Підтримка офлайн-роботи: можна працювати без підключення до центрttps://bing.com/search?q=.

Що відбувається при виконанні команди git add .?¶

Команда git add . додає всі нові або змінені файли у поточному каталозі до індексу (або staging area) для наступного коміту. Це означає, що всі файли, які були змінені або додані у поточному каталозі, будуть включені у наступний комі

Що відбувається при виконанні команди git commit?¶

Команда git commit фіксує зміни в локальному репозиторії Git. Ось основні етапи, що відбуваються при виконанні цієї команди:

Фіксація змін: Git бере зміни, які були додані у "staging area" (за допомогою команди git add), і створює новий коміт. Цей коміт містить копію всіх змінених файлів, які були підготовлені до фіксації.

Ідентифікація коміту: Кожен коміт отримує унікальний хеш (SHA-1), який є ідентифікатором цього коміту. Він забезпечує унікальність коміту і дозволяє Git відслідковувати історію змін.

Коментар до коміту: Користувач повинен додати опис або повідомлення коміту (commit message), яке пояснює, що було змінено. Це допомагає іншим (або вам у майбутньому) розуміти суть змін, зафіксованих у цьому коміті.

Оновлення гілки: Після створення коміту Git оновлює поточну гілку (branch) до цього нового коміту, вказуючи, що тепер гілка містить найновіші зміни.

Якою Git-командою виконується розміщення локальної копії Git-репозиторію проєкта на віддаленому, що знаходиться на GitHub?¶

Завантажити зміни на GitHub: Використовується команда git push, щоб відправити локальні зміни в віддалений репозиторій.
